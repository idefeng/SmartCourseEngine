
### 2. 文件上传服务架构
#### 2.1 核心类设计
```python
# 分片上传管理器
class ChunkedUploadManager:
    - 上传任务管理
    - 分片存储合并
    - 哈希验证
    - 状态持久化

# 文件上传服务
class FileUploadService:
    - 初始化上传
    - 分片上传
    - 完成上传
    - 状态查询
```

#### 2.2 上传流程
```
1. 初始化上传 → 2. 上传分片 → 3. 合并文件 → 4. 完成上传
     ↓              ↓              ↓              ↓
  验证文件       验证分片       计算哈希       更新状态
  计算分片       存储分片       合并文件       清理临时
  创建任务       更新进度       移动文件       发送通知
```

### 3. 实时通信流程
#### 3.1 视频上传和分析流程
```
前端上传 → WebSocket连接 → 初始化上传 → 分片上传 → 完成上传
    ↓           ↓              ↓           ↓          ↓
  进度显示    连接建立       任务创建    进度推送   分析开始
             消息接收       进度跟踪    实时更新   进度跟踪
```

#### 3.2 进度通知机制
```python
# 进度更新
await websocket_service.send_video_upload_progress(
    user_id=user_id,
    task_id=upload_id,
    progress=progress,
    file_name=file_name
)

# 分析开始
await websocket_service.send_video_analysis_started(
    user_id=user_id,
    task_id=task_id,
    video_id=video_id
)
```
