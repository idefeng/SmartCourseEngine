# SmartCourseEngine 前端启动与访问说明

本文档用于在本地完整启动后端服务与管理前端，并通过浏览器访问系统。

## 1. 环境准备

### 必备软件
- Docker Desktop（已启动）
- Docker Compose（随 Docker Desktop 提供）
- Node.js >= 18
- npm >= 9

### 建议检查
```bash
docker --version
docker compose version
node -v
npm -v
```

---

## 2. 启动后端（Docker）

在新终端执行：

```bash
cd /Users/idefeng/DEV/SmartCourseEngine/new_architecture/deploy
docker compose down --remove-orphans
docker compose up -d --build
docker compose ps -a
```

### 预期状态
- `api-gateway`、`video-analyzer`、`knowledge-extractor`、`search-engine`、`video-analysis-worker` 为 `Up`
- `postgres`、`redis`、`rabbitmq`、`neo4j`、`minio` 为 `Up (healthy)`

---

## 3. 配置前端环境变量

前端默认会请求 `http://localhost:8001`，与当前后端网关端口不一致，必须覆写。

在新终端执行：

```bash
cd /Users/idefeng/DEV/SmartCourseEngine/new_architecture/admin-frontend
cat > .env.local << 'EOF'
NEXT_PUBLIC_API_BASE_URL=http://localhost:8000
NEXT_PUBLIC_WS_URL=ws://localhost:8000/ws
EOF
```

说明：
- `NEXT_PUBLIC_API_BASE_URL`：前端调用后端 API 的基地址
- `NEXT_PUBLIC_WS_URL`：WebSocket 地址；若网关未启用 `/ws`，仅实时功能不可用，不影响大多数页面访问

---

## 4. 启动前端（Next.js）

> 当前 Docker 中 Grafana 占用 `3000` 端口，前端建议使用 `3001`。

```bash
cd /Users/idefeng/DEV/SmartCourseEngine/new_architecture/admin-frontend
npm install
npm run dev -- -p 3001
```

看到 `ready - started server on ...:3001` 即表示前端启动成功。

## 5. 访问地址总览

### 业务访问
- 管理前端：`http://localhost:3001`
- API 网关 Swagger：`http://localhost:8000/docs`
- API 网关健康检查：`http://localhost:8000/health`

### 基础设施
- RabbitMQ 管理台：`http://localhost:15672`（admin/admin123）
- Prometheus：`http://localhost:9090`
- Grafana：`http://localhost:3000`（admin/admin123）
- MinIO Console：`http://localhost:9001`（admin/admin123）
- Neo4j Browser：`http://localhost:7474`（neo4j/admin123）

## 6. 快速验收清单

按顺序检查：

1. `docker compose ps -a` 无关键服务 `Exited`
2. 浏览器打开 `http://localhost:8000/docs` 正常
3. 浏览器打开 `http://localhost:3001` 正常加载前端页面
4. 前端页面触发一次 API 请求（查看浏览器 Network）返回 2xx/4xx（非网络失败）
5. 若使用视频分析链路，观察 `video-analysis-worker` 日志有消费记录

---

## 7. 常见问题与处理

### 问题 1：`zsh: command not found: docker`
未安装或未启动 Docker Desktop。安装并启动后重开终端。

### 问题 2：前端启动报端口占用
使用其他端口启动：
```bash
npm run dev -- -p 3002
```
然后访问 `http://localhost:3002`。

### 问题 3：前端页面提示网络错误
优先检查：
```bash
curl -i http://localhost:8000/health
docker compose ps -a
docker compose logs --tail=100 api-gateway
```

### 问题 4：WebSocket 报错
如果接口与页面主流程正常，可先忽略；这是网关未启用 `/ws` 导致的实时通道不可用。

### 停止后端
```bash
cd /Users/idefeng/DEV/SmartCourseEngine/new_architecture/deploy
```
