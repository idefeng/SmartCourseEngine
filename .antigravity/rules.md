# SmartCourseEngine 重构与开发规范 (v1.0)

本项目是基于 AI 驱动的教育科技重构项目，旨在通过大模型能力实现课程内容（教材、PPT、视频、题库）的自动化与半自动化生产。

## 🎨 Vibe Coding 规范 (核心开发理念)

1. **AI 优先设计 (AI-First Design)**:
    - 任何新功能的设计必须优先考虑其 AI 可解释性和自动生成能力。
    - 接口定义需对大模型友好（使用清晰的命名空间和文档说明）。
    - 优先实现 RAG (检索增强生成) 流程，确保 AI 输出内容有据可查。

2. **极高审美标准 (Premium Aesthetics)**:
    - **视觉冲击**: UI 不得使用原生默认组件，必须采用现代 CSS 系列（如 Glassmorphism, 渐变色背景）。
    - **微交互**: 所有的按钮点击、页面切换必须伴随流畅的过渡动画。
    - **中医意韵**: 针对中医领域，色调建议采用温润的中性色结合现代科技感，强调专业与信任。

3. **快速反馈循环 (Fast Feedback Loops)**:
    - 模块化架构：确保 `SmartParser` 和 `VideoGen` 等核心组件可以独立运行和测试。
    - 编写清晰的提示词 (Prompts) 管理机制，支持快速迭代生成效果。

---

## 🔒 隐私与安全规范

1. **数据主权与本地优先**:
    - 用户数据、原始教材文件、以及生成的知识库（Chroma/Milvus）**默认物理存储于本地或企业私有云**。
    - 禁止在未脱敏情况下将包含个人隐私或商业机密的文档上传至公有云大模型 API。

2. **敏感信息脱敏 (Data Masking)**:
    - 在通过 `Unstructured` 解析文档后，调用大模型提取知识点前，必须对电话、身份证号、地址等敏感信息进行占位符替换。

3. **API 与密钥管理**:
    - **严禁**将 API Key、数据库密码、AWS S3 密钥硬编码在代码中。
    - 必须统一通过 `.env` 或系统环境变量读取。
    - 生产环境必须开启 API 请求审计日志。

## 🛠️ 操作指南 (Operation Guide)

1. **环境启动**:
    - 使用 `docker-compose up -d` 启动核心组件（PostgreSQL, MinIO, Chroma）。
    - 本地 Python 环境建议使用 `conda` 或 `venv`，并通过 `pip install -r requirements.txt` 安装依赖。

2. **AI 内容生产流**:
    - **Step 1**: 将课件/讲义放入 `uploads` 目录。
    - **Step 2**: 运行 `python SmartParser/main.py` 进行知识点提取与结构化。
    - **Step 3**: 使用管理后台基于结构化知识生成 PPT 提纲。
    - **Step 4**: 调用数字人渲染 API 生成教学视频。

3. **质量闭环 (QA Loop)**:
    - **AI 生成 + 10% 人工抽查**: 每批次生成内容必须有 10% 的人工硬性审核。
    - **反馈反馈**: 若抽查准确率低于 95%，必须停止流水线，重新优化 Prompt 或微调向量检索策略。
