# SmartCourseEngine 48小时演示版 (MVP) 开发与交付计划

## 🎯 演示目标
在 2 天内向甲方展示一个闭环的 AI 驱动课程生产流程：从**原始中医课件录入**到**结构化知识点提取**，再到**自动生成教材大纲与 PPT 预览**，以此验证方案的可行性与 AI 效率。

---

## 📅 时间表 (48小时倒计时)

### 第一阶段：基础设施与数据准备 (第1-12小时)
**目标**：跑通底层的解析逻辑。

1. **环境一键启动 (第1-3小时)**
   - 编写 `docker-compose.yml` 启动 PostgreSQL (业务数据) 和 ChromaDB (向量数据库)。
   - 配置 `.env` 环境文件，连接 LLM (推荐 DeepSeek 或 Qwen)。
2. **演示素材初始化 (第4-6小时)**
   - 选取一份高质量的中医讲义 (如《中医基础理论》) 放入 `uploads`。
   - 配置 `SmartParser` 的解析规则。
3. **知识提取跑通 (第7-12小时)**
   - 实现文档到 Markdown 的清理。
   - 编写 Prompt 提取 JSON 格式的知识点。
   - **产出**：数据库中出现结构化的中医知识点。

### 第二阶段：核心业务逻辑实现 (第13-24小时)
**目标**：生成演示的核心资产。

1.  **教材大纲生成 (第13-18小时)**
    -   开发基于 RAG 的大纲生成逻辑。
    -   生成配套的教学目标、重难点。
2.  **教学质量评估 MVP (第19-22小时)**
    -   实现简单的语音转写流程 (配合 Whisper API/本地服务)。
    -   编写 Prompt 基于转写文本分析：知识点覆盖度、师生互动频次、口头禅统计。
3.  **演示 PPT 内容生成 (第23-24小时)**
    -   将知识点自动拆分为 PPT 页面内容（标题、要点、配图建议）。
    -   **产出**：一份完整的、由 AI 生成的课程大纲、PPT 文稿，以及一份课堂教学质量分析 JSON。

### 第三阶段：Vibe 设计感前端构建 (第25-36小时)
**目标**：增强视觉冲击力，让甲方觉得“高级”。

   - 展示“知识提取进度条”、“知识图谱简易视图”以及**“教学质量仪表盘”**（含互动率曲线、知识覆盖率雷达图）。
2. **多模态模拟演示 (第31-36小时)**
   - 生成 1-2 张中医相关的 AI 示意图 (DALL-E 3 / Midjourney)。
   - 录制或生成一段 30 秒的数字人讲课 Mock 视频。
   - **产出**：可交互的 Web 演示界面。

### 第四阶段：测试、优化与文档 (第37-48小时)
**目标**：确保演示不翻车，交付代码。

1. **链路压力测试 (第37-40小时)**
   - 确保护路径、数据库连接、API 调用在多轮点击下稳定。
2. **演示文稿准备 (第41-44小时)**
   - 制作一份 5-8 页的 PPT，配合系统演示。
3. **同步推送与文档更新 (第45-48小时)**
   - 最终代码库推送。
   - 更新项目 `README.md`。

---

## 🛡️ 演示关键点 (Tips for Success)
- **不可缺失的“Wow”阶段**：在演示时，必须展示 AI 如何把枯燥的 PDF 一瞬间变成结构化 JSON 和精美 PPT 的过程。
- **降级方案**：如果 API 调用延迟，准备好预先生成的静态数据（Pre-generated Cache）作为备选。
- **安全保障**：演示现场展示本地存储路径，强调隐私合规。

## ⚙️ 技术栈 (MVP 版)
- **后端**: Python (FastAPI)
- **解析**: Unstructured
- **数据库**: PostgreSQL + ChromaDB
- **LLM**: DeepSeek V3 / Qwen-4
- **前端**: React (Vite) / Vue 3
- **视频工具**: 腾讯智影 (API Mock) / MuseTalk
