## 快速部署

### 🚀 一行命令启动

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/SmartCourseEngine.git
cd SmartCourseEngine/SmartParser

# 2. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入 API Keys

# 3. 一键启动
docker-compose up -d
```

### 访问服务

| 服务 | 地址 | 说明 |
|------|------|------|
| Web 界面 | http://localhost:8501 | Streamlit 主应用 |
| 任务监控 | http://localhost:5555 | Flower 监控面板 |
| Redis | localhost:6379 | 消息队列 |

## 详细配置

### 环境变量 (.env)

创建 `.env` 文件：

```bash
# =============================================================================
# SmartCourseEngine 环境配置
# =============================================================================

# ----- API Keys -----
DEEPSEEK_API_KEY=sk-your-deepseek-key
HEYGEN_API_KEY=your-heygen-key

# ----- API 配置 -----
API_BASE_URL=https://api.deepseek.com/v1
MODEL_NAME=deepseek-chat

# ----- 本地 LLM (可选) -----
OLLAMA_HOST=http://ollama:11434
OLLAMA_MODEL=qwen2.5:7b

# ----- Redis -----
REDIS_URL=redis://redis:6379/0
CELERY_BROKER_URL=redis://redis:6379/0
CELERY_RESULT_BACKEND=redis://redis:6379/0
```

### Docker Compose 配置

```yaml
# 标准启动
docker-compose up -d

# 仅启动 Web 应用 (不启动 Worker)
docker-compose up -d app redis

# 启动本地 LLM
docker-compose --profile local-llm up -d

# 启动备份服务
docker-compose --profile backup up -d

# 查看日志
docker-compose logs -f app
docker-compose logs -f worker
```

## GPU 加速

### 安装 NVIDIA Container Toolkit

```bash
# Ubuntu/Debian
distribution=$(. /etc/os-release;echo $ID$VERSION_ID)
curl -s -L https://nvidia.github.io/nvidia-docker/gpgkey | sudo apt-key add -
curl -s -L https://nvidia.github.io/nvidia-docker/$distribution/nvidia-docker.list | \
  sudo tee /etc/apt/sources.list.d/nvidia-docker.list

sudo apt-get update
sudo apt-get install -y nvidia-container-toolkit
sudo systemctl restart docker
```

### 验证 GPU

```bash
docker run --rm --gpus all nvidia/cuda:12.1-base nvidia-smi
```

### GPU 启动

```bash
# 启动带 GPU 支持的服务
docker-compose --profile local-llm up -d
```

## 本地 LLM 部署

### 方式1: 使用 Docker Compose 内置 Ollama

```bash
# 启动 Ollama 服务
docker-compose --profile local-llm up -d ollama

# 拉取模型 (首次需要)
docker exec -it smartcourse-ollama ollama pull qwen2.5:7b

# 测试
docker exec -it smartcourse-ollama ollama run qwen2.5:7b "你好"
```

### 方式2: 独立部署 Ollama

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen2.5:7b
ollama pull llama3:8b

# 启动服务 (默认端口 11434)
ollama serve
```

### 推荐模型

| 模型 | 大小 | 推荐场景 |
|------|------|----------|
| qwen2.5:7b | 4.7 GB | 中文课件生成 (推荐) |
| qwen2.5:14b | 9.7 GB | 高质量中文生成 |
| llama3:8b | 4.7 GB | 英文内容 |
| mistral:7b | 4.1 GB | 通用场景 |

### 在应用中使用本地 LLM

在 Web 界面的侧边栏配置中：
1. 勾选 "使用本地 LLM"
2. 填入 Ollama 地址 (默认: http://localhost:11434)
3. 选择模型

## 数据备份与恢复

### 手动备份

```bash
# 进入容器执行备份
docker exec smartcourse-app python scripts/backup.py

# 或在宿主机执行
python scripts/backup.py
```

### 自动备份

启动备份服务 (每天自动备份):

```bash
docker-compose --profile backup up -d backup
```

### 恢复备份

```bash
# 列出可用备份
python scripts/backup.py --list

# 恢复指定备份
python scripts/backup.py --restore backups/backup_20260131_120000.zip
```

### 备份内容

| 目录/文件 | 说明 |
|-----------|------|
| chroma_db/ | 向量数据库 |
| output_markdown/ | 解析的文档 |
| standards_library/ | 行业标准库 |
| generated_courseware/ | 生成的课件 |
| config.json | 配置文件 |

## 常见问题

### Q: 启动失败，提示端口被占用

```bash
# 检查端口占用
netstat -tlnp | grep 8501

# 修改 docker-compose.yml 中的端口映射
ports:
  - "8502:8501"  # 改用其他端口
```

### Q: GPU 不可用

```bash
# 检查 NVIDIA 驱动
nvidia-smi

# 检查 Container Toolkit
docker run --rm --gpus all nvidia/cuda:12.1-base nvidia-smi

# 确保 docker-compose 使用正确的 profile
docker-compose --profile local-llm up -d
```

### Q: Celery Worker 无法连接 Redis

```bash
# 检查 Redis 是否运行
docker-compose ps redis

# 检查网络
docker network ls
docker network inspect smartparser_smartcourse-network
```

### Q: 内存不足

```bash
# 限制容器内存
# 在 docker-compose.yml 中添加:
deploy:
  resources:
    limits:
      memory: 4G
```

### Q: 如何升级模型?

```bash
# Ollama 模型升级
docker exec -it smartcourse-ollama ollama pull qwen2.5:7b

# 或使用新模型
docker exec -it smartcourse-ollama ollama pull qwen2.5:14b
```

## 生产环境建议

### 安全加固

1. **修改默认端口**
2. **配置 HTTPS** (使用 Nginx 反向代理)
3. **设置防火墙规则**
4. **定期更新镜像**

### Nginx 反向代理配置

```nginx
server {
    listen 80;
    server_name your-domain.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:8501;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 日志管理

```bash
# 配置日志轮转
docker-compose logs --tail=1000 app > logs/app.log
```
